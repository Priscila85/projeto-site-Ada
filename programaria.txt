Ada Lovelace
Augusta Ada King, condessa de Lovelace, nasceu em Londres em 1815, mais conhecida como Ada Lovelace, foi a primeira pessoa a escrever um algoritmo para ser processado por uma máquina na história da computação.

Infância
Seu pai era o poeta romântico Lord Byron, conhecido por seus exageros como ter um urso domesticado em seu quarto e por atos (incomuns para a época), como se relacionar com homens e mulheres. Sua mãe era a rica e culta Annabella Milbanke, apaixonada por matemática e pouco paciente com as maluquices do marido.

Lord Byron bebia demais o que resultou no divórcio quando Ada tinha só 5 semanas de vida. Annabella foi mãe solo preocupada em garantir que Ada não ficasse parecida com o pai. Investiu na educação rigorosa da menina contratando os melhores professores. Acreditava que, se a filha se interessasse por matemática e ciências, se afastaria da poesia e das tendências excêntricas herdadas do pai.

Ada era fascinada por matemática e poesia (para desespero de sua mãe). Aos doze anos, encantou-se pela engenharia mecânica e escreveu um livro chamado Flyology (ou Voologia), com desenhos de um plano para a construção de uma máquina de voar. Ao contar para sua mãe sobre a máquina, Annabelle achou que Ada estava se desviando dos estudos e a reprimiu. Então, pediu para que Ada focasse em se tornar uma dama.

Juventude
Aos 17 anos, Ada foi apresentada à corte e passou a ser convidada para festas na cidade. Seu jeito inteligente e rápida atraiu muitos solteiros, mas ela se interessou por um velho matemático chamado Charles Babage, que lhe contou sobre a invenção que trabalhava há anos, a “máquina diferencial”, que fazia grandes cálculos.

Charles gostou da jovem, ela era uma das poucas pessoas que se interessou e entendeu suas invenções. Quando Ada conheceu a máquina diferencial, viu nela muitas possibilidades que nem mesmo Charles havia pensado. O entusiasmo da jovem matemática o impressionava.

Vida adulta
Charles era viúvo e lutava por investimentos para os seus projetos. Ada se casou e teve 3 filhos. Em 1830, isso significava que ela deveria abandonar os estudos e se dedicar aos filhos. Trocava cartas com Charles, acompanhava o trabalho dele de longe. Tinha pouco tempo para os próprios estudos. Passou a resolver problemas matemáticos à noite, após as crianças dormirem. Ela queria era trabalhar com Charles em suas máquinas incríveis.

Tempos depois, Charles criou o Engenho Analítico, melhor do que a máquina diferencial, pois seria capaz de analisar dados e guardar informações na memória. Algo tão revolucionário que poucas pessoas entenderam. Para divulgar sua nova máquina, precisaria publicar artigos em francês, língua que ele não sabia, mas Ada era fluente. Como ela entendia as criações de Charles, era a pessoa perfeita para traduzir esses artigos.

Enquanto Ada traduzia o conteúdo, fazia anotações por conta própria, imaginava inúmeras possibilidades que a máquina teria. Não tardou para que suas anotações ficassem maiores do que o próprio artigo.

Charles queria que a máquina resolvesse problemas matemáticos. Ada pensava em algo muito maior: os números poderiam representar coisas totalmente diferentes como letras e notas musicais. Escreveu instruções detalhadas com diagramas e tabelas explicando como o engenho analítico deveria processar uma equação matemática complicada. Assim, a máquina resolveria em segundos o que um humano levaria horas para solucionar. Os computadores ainda não existiam, mas Ada tinha acabado de criar o primeiro programa de computador. Ada não viveu para ver que suas ideias se transformariam nos computadores modernos que usamos hoje.

Como vocês viram, para construir a estrutura HTML de um formulário, é preciso:

a própria tag de formulário <form>, 
uma tag <input> para cada campo , 
definir que tipo de dado aquele campo vai receber (atributo type do <input>);
incluir um rótulo para as pessoas visitantes saberem que dado é para colocar naquele campo (tag <label>).
identificar a qual campo o rótulo se refere com o atributo for do <label>, que recebe como valor a id do campo (<input>)
o botão (tag <button>) para o comportamento de enviar as informações.

👩‍🏫 Quando estamos começando, a gente tem a necessidade de firmar bem os conceitos - isso é normal! Mais importante do que decorar os itens, é entender o funcionamento e comportamento de cada item! 

 

Por que precisamos criar vínculo entre a tag <label> e a tag <input>?
Apesar de elas estarem próximas uma da outra, nada no nosso código associa o <label> ao <input>, o navegador e outros mecanismos que vão ler esse código, como leitores de tela ou robôs de busca, não têm como saber dessa correspondência - lembra que sistemas não têm poderes de adivinhação! 

Por isso é uma boa prática indicar no elemento <label> a qual campo (<input>) esse rótulo se refere. Fazemos isso por meio do atributo for, que deve receber o valor do identificador do campo correspondente (para isso, precisamos criar esse identificador no campo <input>). Já aprendemos como criar identificadores, por meio do atributo id! :)


Além de obtermos o comportamento que a Dani mostrou no vídeo, de clicar na <label> e o cursor ir direto para o campo correspondente, como se aumentasse a área clicável do campo, isso deixa o código mais legível e melhora a usabilidade do formulário. Quer mais um motivo? É uma das Diretrizes de Acessibilidade para Conteúdo Web e está na versão 2.1 (WCAG), pois facilita a acessibilidade do formulário para pessoas que usam leitores de tela, por exemplo.

 

Vale o lembrete: por enquanto, estamos construindo apenas a estrutura do nosso formulário como HTML. Depois cuidaremos do estilo e também das interações que podem ser feitas com ele.

Agora que já sabemos o básico do Git e seu terminal, vamos pro GitHub🥳

Entendendo sobre GitHub

Você se lembra de quando a Pachi disse que o GitHub é uma plataforma online que funciona como uma estante virtual para projetos que usam Git?  Ele não é o único site que oferece esse serviço, mas ele é o mais usado pelo mundo afora🌏

Ele oferece vários recursos gratuitos para aprender e praticar. No futuro, ele também será útil para mostrar seus projetos em um portfólio online🖥️

➡️Acesse github.com

Clique em Sign Up (Cadastrar-se) > Insira seu e-mail (o mesmo do Git) > crie uma senha e escolha um nome de usuário que será seu @ no github.

Depois de criar a conta temos que confirmar o e-mail para ativar ela, então verifique a sua caixa de entrada e clique no link de ativação.

Conectando Git e Github

Como mencionado, o git vive no seu computador, mas a gente quer compartilhar nosso código com o mundo, então vamos criar uma ponte entre os dois🌉

➡️O primeiro passo é criar uma chave SSH. A chave SSH é uma chave de segurança que permite que o Git no computador se conecte ao GitHub de forma segura, sem precisar inserir senha toda hora.

No terminal (Git Bash), digite: ssh-keygen -t ed25519 -C "seu-email@email.com"

(substitua o e-mail pelo usado na conta do GitHub)

Pressione Enter para aceitar os padrões. 

Agora que temos essa chave do nosso Git, precisamos adicionar ele no nosso GitHub:

➡️No terminal vamos digitar cat ~/.ssh/id_ed25519.pub e copiar o texto que será  exibido.

No GitHub Vá em Settings > SSH and GPG Keys > New SSH Key.

Vamos colar nossa chave aqui e salvar. Se tudo deu certo, seu Git e Github estão conectados..

➡️Vamos checar se deu certo?

No terminal, digite: ssh -T git@github.com

O GitHub deve responder algo como:

"Hi [seu usuário]! You've successfully authenticated, but GitHub does not provide shell access."

Essa mensagem basicamente significa que deu tudo certo e agora nosso Git está oficialmente conectado ao nosso GitHub!

Aprendendo sobre repositório

Um repositório é como uma pasta especial onde o Git registra tudo que acontece no projeto e te permite organizar e controlar o histórico de alterações📁

Vamos criar um repositório pelo git na nossa máquina e depois enviar ele pro Github conectando os dois. 

➡️Começando pelo Git, vamos abrir o terminal e navegar até a pasta onde queremos criar nosso repositório.

cd programapasta
Se essa pasta ainda não existir, podemos criar ela pelo Git com o comando mkdir, (make directory, ou criar diretório em português)

Primeiro se certifique que você está no local certo com pwd e então digite mkdir programapasta
➡️Dentro da nossa pasta, vamos digitar git init
Agora vamos criar um arquivo dentro desta pasta com git add index.html

git add é o comando que cria um arquivo, e index.html é o nome do arquivo.
Sempre que mudarmos algo no git, precisamos salvar a mudança com um commit, que é um registro da mudança e do estado atual do projeto.
Em seguida, digite: git commit -m "Primeiro commit: adicionando index.html"
entre aspas você deve colocar uma curta descrição da atualização feita.
➡️Vamos conectar essa pasta ao nosso GitHub:
No GitHub, clique em New Repository (Novo Repositório) e vamos preencher:
-Nome: Nome único para o repositório, vamos usar meu primeiro repo
-Descrição: Aprendendo sobre repositórios
-Visibilidade: Público ou privado. Vamos deixar público pra poder compartilhar com o mundo!
⚠️Não marque a opção de criar README ou adicionar arquivos.
Clique em Create Repository.
Com isso a gente criou dentro do GitHub um espaço para trazer nosso projeto que está no Git.
Seguindo no terminal
De volta ao terminal, vamos adicionar o repositório remoto ao nosso repositório local:
➡️Digite: git remote add origin git@github.com: seu-usuario/seu-repositorio.git
➡️Podemos confirmar se a conexão foi feita com git remote -v
➡️Agora que fizemos a conexão podemos enviar o Projeto para o GitHub.
No terminal digitamos git push -u origin main
Isso manda todos os arquivos para o GitHub.
Para verificar se deu certo, acesse seu repositório no site do GitHub e veja se o arquivo está lá!
